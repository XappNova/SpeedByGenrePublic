Frequently Asked Questions

Why isn't the playback speed changing?

A few things to check: make sure the extension is enabled in Chrome's toolbar. Try refreshing the YouTube page after enabling it. If the video is an ad, speed changes are intentionally disabled. Also confirm that you've set a speed rule for that video's category in the extension popup.

How does the extension detect the video category?

SpeedByGenre reads the category label that YouTube displays on the video page (e.g., "Music", "Gaming", "News & Politics"). This is the same category the video uploader assigned when publishing. No external API calls are made — it's all read locally from the page.

Can I set a custom speed for a specific channel?

Currently, rules are set per category (e.g., all Music videos at 1x). Channel-specific or creator-specific rules are planned for a future update. If this is important to you, let us know at support@xappnova.com — user feedback shapes our roadmap.

Will my settings sync across devices?

Yes. Your speed preferences are saved in chrome.storage.sync, which Chrome synchronizes across all devices where you're signed into the same Google account. No XAppNova server is involved — the sync is handled entirely by Chrome.

Does the extension slow down YouTube or affect page performance?

No. SpeedByGenre is lightweight and only activates on YouTube pages. It reads a single text element from the page and sets the video playback rate — both are near-instant operations with no measurable impact on page load or performance.

Does it work with YouTube Premium or YouTube Music?

SpeedByGenre works on the standard YouTube website (youtube.com). YouTube Music (music.youtube.com) is a separate app and is not currently supported. YouTube Premium subscribers will find the extension works as normal on the main YouTube site.

How do I reset my settings to the defaults?

Open the extension popup, adjust each category speed back to your preferred value, or uninstall and reinstall the extension to start fresh. All settings are stored locally and removed when the extension is uninstalled.
